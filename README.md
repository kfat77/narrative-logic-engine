# Narrative Logic Engine

一个为叙事、立场表达与论证分析而做的 Codex Skill。它把“结论先行”的灵感整理为可检查的前提、推演和成文稿：追求内部自洽，但不把自洽当成事实。

## 适合谁

- **科幻小说作者、编剧与世界观构造者**：用一条明确的反事实公理推出稳定的技术、制度与人物冲突。
- **粉丝文化、公共关系与社群叙事的观察者**：分析某种价值框架怎样重设“好/坏”的衡量标准，并写出透明的立场文本。
- **民科式论证、狂热社群话语与自封闭观点的研究者**：识别循环论证、反证豁免与伪术语如何运作，明确什么证据能让结论被修正。
- **需要写作立场文章的人**：将价值判断、事实主张和待证假设分开，形成可被公平质疑的论证。

它**不适合**把未经证实的理论伪装成科学事实、替真实人物组织网暴或攻击、虚构数据与权威背书，或设计操控和欺骗话术。

## 输出内容

默认的“混合模式”会依次给出：

1. 现实标签与适用边界
2. 三条基础前提
3. 工作词汇表
4. 推演链与限制条件
5. 最强异议及公平回应
6. 可修正条件
7. 一篇可直接使用的成文稿
8. 事实、假设、争议解释与虚构元素的现实检验

## 安装

将整个目录复制到 Codex Skills 目录：

```text
%USERPROFILE%\\.codex\\skills\\narrative-logic-engine
```

然后重启或刷新 Codex 的 Skill 列表。

## 使用

```text
使用 $narrative-logic-engine：
核心结论：月相会让固体接触摩擦暂时消失。
领域：硬科幻世界观设定。
现实性标记：虚构。
输出模式：混合。
语气与长度：设定集风格，约 800 字。
```

也可使用 `draft`（只要成文稿）或 `toolkit`（只要结构化骨架与可复用表达）。

## 设计原则

- 明示前提，不偷渡定义。
- 钢人化反方，而非攻击质疑者。
- 对真实议题保留可证伪性与来源要求。
- 对虚构议题要求设定内一致性。
