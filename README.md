# Narrative Logic Engine

一个为叙事、立场表达与论证分析而做的多模态 Codex Skill。它会吸收用户在当前任务中提供的文字、图片、视频、字幕、转写和笔记，建立可追溯的资料台账与世界观，再把它们整理为可检查的前提、推演和成文稿：追求内部自洽，但不把自洽当成事实。

## 适合谁

- **科幻小说作者、编剧与世界观构造者**：用一条明确的反事实公理推出稳定的技术、制度与人物冲突。
- **粉丝文化、公共关系与社群叙事的观察者**：分析某种价值框架怎样重设“好/坏”的衡量标准，并写出透明的立场文本。
- **民科式论证、狂热社群话语与自封闭观点的研究者**：识别循环论证、反证豁免与伪术语如何运作，明确什么证据能让结论被修正。
- **需要写作立场文章的人**：将价值判断、事实主张和待证假设分开，形成可被公平质疑的论证。

它**不适合**把未经证实的理论伪装成科学事实、替真实人物组织网暴或攻击、虚构数据与权威背书，或设计操控和欺骗话术。

## 输出内容

对新上传的多模态资料，如未指定产物，默认先给“世界观圣经”以便确认 canon；选择“混合模式”时会依次给出：

1. 资料台账：原始观察、推测、置信度与待解决问题
2. 世界观圣经：实体、关系、规则、时间线、矛盾与 canon 状态
3. 现实标签与适用边界
4. 三条基础前提及其来源
5. 工作词汇表
6. 推演链与限制条件
7. 最强异议及公平回应
8. 可修正条件
9. 一篇可直接使用的成文稿
10. 事实、假设、争议解释与虚构元素的现实检验

### 多模态资料的处理

- **文字/文档**：抽取事实主张、角色、事件、规则与时间线。
- **图片**：分离画面中可直接观察到的内容，与对风格、身份、时代或因果的推测。
- **视频**：使用可读取的字幕、转写、时间戳与代表帧；若无法直接访问视频内容，会请求转写或带时间戳的截图，不会臆造未看到的场景。

每个关键结论都标记为“原始来源”“支持性推论”“工作假设”或“创作补充”，并带有页码、时间戳或图像区域等来源定位。概念图默认只证明画面中可见的设计，不自动证明其机制或世界内事实。Skill 会在当前会话及所附资料范围内建立模型；它不会声称在会话之外永久记住或自行训练这些资料。

## 安装

将整个目录复制到 Codex Skills 目录：

```text
%USERPROFILE%\\.codex\\skills\\narrative-logic-engine
```

然后重启或刷新 Codex 的 Skill 列表。

## 使用

```text
使用 $narrative-logic-engine：
核心结论：月相会让固体接触摩擦暂时消失。
领域：硬科幻世界观设定。
现实性标记：虚构。
输出模式：混合。
语气与长度：设定集风格，约 800 字。

资料：以下三篇设定笔记、两张概念图和一段带字幕的城市宣传片都是当前 canon。
```

也可使用 `world-bible`（只要资料台账与可复用世界观）、`draft`（只要成文稿）或 `toolkit`（只要结构化骨架与可复用表达）。

## 设计原则

- 明示前提，不偷渡定义。
- 钢人化反方，而非攻击质疑者。
- 对真实议题保留可证伪性与来源要求。
- 对虚构议题要求设定内一致性。
